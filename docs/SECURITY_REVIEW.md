# SECURITY_REVIEW — v1.1.0 (v1.0.0 기준 갱신)

v0.1.1의 보안 설계를 유지하며, 추가 기능(해시/이력 DB/공유 수신, v1.1.0의 완전 자동화 오케스트레이터)에서도
데이터 외부 유출 경로를 만들지 않았습니다.

## 0. v1.1.0에서 새로 검토한 위험
- **완전 자동화 확대**: 접근성 서비스가 스스로 여러 단계를 순차 조작하게 되므로, 오케스트레이터가
  `READY_FOR_USER` 이후 절대 진행하지 않도록 상태머신 레벨에서 강제했다(`PipelineState.isHalting`).
- **문맥별 안전 클릭**: "확인"/"완료" 같은 범용 단어를 화면별로 제한 허용하게 되면서, 금지어 근접 검사에
  더해 클릭 직전 `screen_role` 이 현재 화면과 정확히 일치하는지 재검사하는 계층을 추가했다(`safeClick()`).
- **선택자 JSON 신뢰 경계**: 사용자가 SAF로 불러오는 JSON은 스키마 검증을 통과해야만 적용되고, 시험
  적용 → 정식 적용 전 자동 백업(이전 정상본) 구조로 되돌릴 수 있다. 여전히 인터넷 자동 다운로드는 없다.
- **사진 선택기 패키지 확장 최소화**: 실기에서 확인하기 전까지 `PHOTO_PICKER.package_names` 를 비워 두어
  "모든 앱 허용" 상태로 출시되지 않도록 기본값 자체를 안전하게 설계했다.

## 1. 네트워크 격리 (핵심)
- **INTERNET 권한을 선언하지 않습니다.** 앱 프로세스는 어떤 서버로도 소켓 연결/전송을 할 수 없습니다.
- 글·이미지·태그·계정정보·이력의 외부 전송 경로가 원천적으로 존재하지 않습니다.
- 네이버 열기는 다른 앱(네이버 블로그 앱)에 Intent 를 넘기는 방식이라 INTERNET 이 불필요합니다.
- 결과적으로 원격 자동 업데이트/원격 선택자 다운로드도 제공하지 않습니다(수동 SAF 파일 교체만).

## 2. 계정/비밀 정보 미취급
- 네이버 로그인 정보를 저장/입력/요청하지 않습니다. 로그인은 사용자가 직접 합니다.
- 소스 어디에도 API 키/시크릿/비밀번호가 없습니다.
- release 서명키/비밀번호는 소스에 포함하지 않고 **환경변수(GitHub Secrets)로만** 주입합니다.

## 3. 접근성 서비스 최소 권한
- 조작 대상 패키지를 **`com.nhn.android.blog` 1개로만 제한**(`accessibility_service_config.xml`
  의 `packageNames`, v1.0.1부터 네이버 일반 앱/검색 앱은 완전히 제외됨). 사진 선택기 패키지는 실기
  확인 전까지 추가하지 않는다(위 0번 항목).
- 좌표 제스처 미사용(`canPerformGestures=false`), 단일 활성 창만 사용.
- **발행/등록/완료/게시/공개/저장/임시저장/예약/확인** 계열 버튼은 텍스트 목록 + 화면 문맥(screen_role)
  이중 검사로 클릭을 차단한다. 텍스트가 같아도 화면 ID·선택자 역할이 일치하지 않으면 클릭하지 않는다.
- 자동 조작 전 현재 화면이 허용된 네이버 패키지인지 매번 확인. 아니면 조작하지 않고 NeedsUser로 정지하거나
  클립보드 fallback.

## 4. 클립보드 취급(조건부 정리)
- 자동 입력 실패 시 클립보드 복사로 대체하되, **앱이 복사한 값이 현재 클립보드와 동일할 때만** 지웁니다.
- 사용자가 그 사이 다른 것을 복사했거나 클립보드를 읽지 못하면 지우지 않습니다(데이터 보존/안전).

## 5. 저장/백업 격리
- `allowBackup=false` + `data_extraction_rules` 로 root/file/database/sharedpref/external 을 클라우드
  백업·기기이전에서 모두 제외. 자동화 세션 저장(SharedPreferences)도 이 규칙에 포함된다.
- 이력 DB/로그/설정/세션 스냅샷은 앱 내부에만 저장. 로그·세션 저장 어디에도 태그/제목/본문 원문을 남기지
  않는다(태그는 "3/8" 형태 진행률만, 본문/제목은 존재 여부와 길이 비교만).
- 이미지 저장은 MediaStore(scoped storage) 사용 → WRITE_EXTERNAL_STORAGE 등 광범위 권한 불필요.

## 6. ZIP 파싱 방어 (자원 제한, v0.1.1 이식)
- 압축폭탄(과대 용량/고압축률), 다량 엔트리, 경로 탈출(basename 만 사용), 제어문자 파일명, 중복 basename 을 방어.
- 손상/비정상 ZIP 은 앱을 죽이지 않고 사용자 메시지 예외로 안전 중단.

## 7. 권한 목록(최소)
`SYSTEM_ALERT_WINDOW`(플로팅), `FOREGROUND_SERVICE`(+`SPECIAL_USE`), `POST_NOTIFICATIONS`(알림). 그 외
저장/전화/위치/연락처 권한 없음.

## 8. 잔여 위험 / 사용자 책임
- 접근성 서비스는 강력한 권한이므로, 사용자가 신뢰하는 화면(네이버 블로그 글쓰기)에서만 사용해야 합니다.
- 발행/임시저장은 항상 사용자가 직접 확인 후 눌러야 합니다. 앱은 이를 대신 누르지 않습니다(코드 레벨 강제 +
  런타임 문맥 검사 이중 방어).
- **네이버 블로그의 이용약관/자동화 정책 준수는 사용자의 책임**입니다. 이 앱은 접근성 서비스 기반의 UI
  자동 조작 도구이며, 네이버의 공식 API가 아닙니다. 대량/반복 게시가 플랫폼 정책에 저촉될 수 있는지는
  사용자가 직접 확인해야 합니다.
- Google Play Protect 통과 및 실기기 동작은 **미검증**(실기기 검증 필요, BUILD_STATUS 참고).
