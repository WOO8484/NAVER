# 설치·실행 문제 해결 — v1.1.0

## 설치가 안 될 때
- **서명 충돌**: 기존에 다른 서명(예: 로컬 debug 키)으로 설치된 앱이 있으면 같은 패키지명이라도
  설치가 거부됩니다. 기존 앱을 삭제한 뒤 다시 설치하거나, 항상 같은 release 서명 키를 사용하세요.
- **출처를 알 수 없는 앱 설치 차단**: 설정 > 보안 > "출처를 알 수 없는 앱" 허용이 필요할 수 있습니다.

## 네이버 블로그 앱이 안 열릴 때
- [네이버 블로그 앱 열기]를 눌렀는데 반응이 없으면 `com.nhn.android.blog` 가 설치되어 있는지 확인하세요.
  앱이 없으면 설치 안내 팝업이 표시되고, [설치 화면 열기]를 누르면 Play 스토어로 이동합니다.

## 전체 자동 실행 버튼이 비활성화되어 있을 때
③ 전체 자동 실행 카드의 "부족한 항목" 문구를 확인하세요. 주로 다음 중 하나입니다.
- 정상 파싱된 자료 없음 → ZIP을 먼저 불러오세요.
- 이미지 검증 오류 / 저장된 이미지 없음 → ④ 카드에서 "이미지 갤러리에 다시 저장"을 눌러보세요.
- 접근성/오버레이/알림 허용 필요 → ② 권한 카드에서 각각 설정하세요.
- 네이버 블로그 앱 미설치 → 설치 후 다시 시도하세요.
- 다른 자동화 세션 실행 중 → 진행 중인 세션을 [즉시 중지]하거나 완료(발행완료/임시저장/취소 기록) 후
  다시 시도하세요.

## 자동 실행이 계속 일시정지될 때
플로팅 컨트롤의 상태 문구에 표시되는 "사유"를 확인하세요. 대부분은 다음 중 하나입니다.
- 선택자가 실제 네이버 화면과 맞지 않음(선택자 값이 아직 실기 검증 전 — `SELECTOR_RULES.md` 참고).
- 사진 선택기 패키지가 `selector_rules.json` 에 등록되지 않음(기본값은 비어 있음 — 의도된 안전장치).
- 게시판을 자동으로 찾지 못함 — 안내대로 직접 선택하면 이어서 진행됩니다.

이런 경우 ⑤ 선택자 규칙 카드에서 실기 확인한 값으로 JSON을 [시험 적용] → 결과 확인 → [정식 적용]
순서로 교체하세요.

## 이 문서에서 다루지 않는 것
실제 기기별(특히 Samsung SM-G996N 이외 기종) 세부 동작 차이는 이번 버전에서 검증하지 못했습니다
(`미검증-실기기필요`). 문제가 재현되면 ⑤ 카드 아래 진단 로그(발행/게시 관련 원문은 기록되지 않음)를
참고해 선택자를 조정해 주세요.
