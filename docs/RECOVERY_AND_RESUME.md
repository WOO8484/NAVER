# 오류 감지 · 자동 복구 · 이어하기 — v1.1.0

## 1. 감지 범위
- 화면 이탈/다른 앱 전환/네이버 앱 종료: 각 자동화 단계가 스스로 현재 화면 패키지·화면 ID를
  확인하고, 기대와 다르면 `Retryable`/`NeedsUser` 를 반환해 다음 틱에서 자연히 멈춘다.
- 화면 꺼짐: `SafetyMonitor` 가 `ACTION_SCREEN_OFF` 브로드캐스트를 받아 오케스트레이터를 즉시 일시정지한다.
- 접근성/오버레이 권한 해제: `SafetyMonitor.checkPermissionsStillGranted()` 를 상태 갱신 시 주기적으로
  호출해 실행 중 권한이 꺼지면 즉시 일시정지한다.
- 전화 수신/화면 잠금 중 앱 전환도 위 "다른 앱 전환" 감지 경로로 함께 잡힌다(READ_PHONE_STATE 등
  추가 권한 없이, 화면 패키지 확인만으로 처리 — 권한 최소화 원칙).

## 2. 복구 원칙
- 오류 발생 즉시 현재 단계를 저장한다(`AutomationOrchestrator.onUpdate` → `AutomationSessionStore.save()`).
- 다음 단계로 자동 진행하지 않는다(`NeedsUser`/`Blocked`/타임아웃/재시도초과는 모두 즉시 정지).
- 재시도 전 이미 완료된 입력을 확인해 중복 입력을 방지한다(제목/본문은 값 비교, 태그는 커서 위치 기준).
- 자동 복구 불가 시 `PAUSED` 로 남고, 사용자에게 필요한 동작 1개만 짧게 안내한다.

## 3. 진행 상태 영구 저장
`AutomationSessionStore` 가 SharedPreferences에 다음을 저장한다: `session_id`, ZIP SHA-256, 현재 단계,
재개 단계, 완료 단계 목록, 현재 태그 인덱스, 선택 완료 사진 수, 저장 이미지 manifest, 선택 게시판 키,
재시도 횟수, 일시정지 이유, 시험 모드 여부, 최종 갱신 시각. **본문/제목 원문과 계정 정보는 저장하지 않는다** —
원문은 사용자가 같은 ZIP을 다시 불러올 때만 재파싱으로 복원한다.

## 4. 앱 재실행 시 동작
`MainActivity.maybeOfferSessionResume()` 이 시작 시 중단 세션 유무를 확인해 다음 세 가지를 제공한다.
- **이어하기(자료 다시 불러오기)**: 같은 ZIP을 다시 선택 → SHA-256이 일치할 때만 상태를 복원.
  자료 해시가 다르면 복원하지 않고 새로 시작한다(지시서 10.3: 해시 재검증).
- **처음부터 다시**: 저장된 세션을 지우고 잠금을 해제한다.
- **취소**: 아무 것도 하지 않고 다음에 다시 묻는다.
- 몰래 자동 재개하지 않는다(항상 사용자 선택 대기).

## 5. 즉시 중지
`AutomationOrchestrator.cancelNow()` 가 `running=false` 로 전환해 예약된 다음 틱을 실행하지 않고,
상태를 `CANCELLED` 로 저장한다. `StepTimeoutController` 는 단일 토큰만 유지하므로 다음 예약이 잡히지
않으면 그걸로 종료된다(예약 누적/중복 실행 없음). 발행/임시저장 화면은 건드리지 않는다.

## 6. 검증 상태
- 상태 저장/복원 데이터 구조, 이어하기 판정 로직, 단일 세션 잠금: `PASS-실행검증`(단위 테스트).
- SharedPreferences 실제 저장/로드, 프로세스 강제 종료 후 실제 이어하기, 화면 잠금/앱 전환 실기 확인:
  `미검증-실기기필요`(Context/Android 프레임워크 필요 — Robolectric 미사용, 이 환경엔 네트워크가 없어
  의존성을 추가로 내려받을 수도 없었음).
